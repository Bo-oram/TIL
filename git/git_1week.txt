git - 가장 널리 쓰이는 버전관리 도구

◎ 1주차 주요 개념 키워드

→ 버전 관리
: 프로젝트의 변경사항을 파악하고 있다는 것으로
: git은 commit을 사용해서 버전을 관리할 수 있는 도구이다

→ git 초기화
: 내 컴퓨터에 있는 프로젝트 파일을 git이 관리할 수 있게 설정하는 것

→ commit
: 작업한 프로젝트를 저장하는 것
: commit에는 누가, 언제, 무엇을(커밋 메세지)를 포함한다

→ add (혹은 staging, 스테이징)
: 저장할 프로젝트를 선택하는것(커밋할 파일선택)

→ commit history
: 커밋한 프로젝트의 역사 리스트

→ git 첫 작업 순서
: 깃 초기화 -> add -> commit

→ repo
: git이 관리할 수 있도록 설정해 둔 프로젝트를 말하며 repository의 약자인 repo로 부른다
: 내 컴퓨터에 있는 프로젝트 파일을 local repo라고 하며, 다른 곳에서 접속할 수 있는 공간에 있는
프로젝트 파일을 remote repository라 한다

→ Tracking(추적)
: 로컬에서 원격으로 연결하는 것을 트래킹한다라고 한다

→ push
: 로컬의 커밋들을 원격에 반영하는 것

→ pull
: 원격의 커밋들을 로컬에 반영하는것
: clone이란 원격repo만 존재할경우 나의 local repo로 복사해 오는 것을 말한다
