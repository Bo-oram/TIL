◎ 2주차 주요 개념 키워드

→ 협업의 단계
: 1단계 - 문제점을 파악하고 누가 해결할지 정한다
: 2단계 - 각자의 공간에서 작업한다
: 3단계 - 각자 작업한 내용을 병합한다

→ issue
: 협업의 단계에서 1단계에 해당하는 것으로 팍악한 문제점을 누가 해결할지 할당하는 부분이다

→ branch
: 메인 커밋에서 갈라저 나의 공간에서 작업할 수 있다
: 작업할 브랜치를 활성화 하는 것을 체크아웃한다 라고 표현한다
: 체크아웃 후 작업하면 해당 브랜치에만 반영된다

→ merge
: 각 브랜치의 작업내용을 다른 프랜치로 병합하는것, 합치는 것을 머지라 부른다
: 개발 시 기준이 되는 기본 브랜치를 정하고 작업한 브랜치를 기본 브랜치에 합친다
: 브랜치명은 규칙을 가지고 이름 지어야 프로젝트 관리가 용이해진다

→ Merge conflict
: 병합충돌
: 혹 같은 파일에서 동일한 부분이 수정되었을 경우 git은 우리에게 어떤 부분을 반영할것인지를 묻는다
: <<<< HEAD, ===, >>> 등을 삭제하고 반영할 내용을 고친후에 머지 커밋을 하여 해결한다
: 이때 새로운 내용은 추가하지 않는다 혹 추가할 내용이 있다면 머지 커밋 후 다시 커밋한다

→ tracking
: 트래킹 한다는것은 브랜치를 기준으로 원격리포와 로컬리포를 연결하는 것을 말한다
